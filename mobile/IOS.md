# Royal Poker Match — iOS

Yes — this game can ship as an iOS app. It already uses **Capacitor** (same approach as Android). The Xcode project lives at `frontend/ios/`.

## Quick start

| | |
|--|--|
| **App ID** | `com.royalpokermatch` |
| **API** | `https://royal-match-poker.onrender.com` |
| **Project** | `frontend/ios/` |
| **Same as** | Android APK — one React game, native shell |

### What you need (Apple only)

1. A **Mac** with **Xcode** (can’t build or submit from Windows)
2. **CocoaPods** — then `pod install` inside `frontend/ios/App`
3. **Apple Developer Program** ($99/year) for TestFlight / App Store

### On a Mac

```bash
cd frontend
npm install
npm run cap:sync:ios
npm run cap:open:ios
```

In Xcode: pick your **Team** under Signing, then **Run ▶** (simulator or device).

Or:

```bash
npm run cap:run:ios
```

### After game code changes

```bash
cd frontend
npm run cap:sync:ios   # or: npm run cap:sync  (Android + iOS)
```

### App Store / TestFlight (short)

1. Apple Developer → App ID `com.royalpokermatch`
2. App Store Connect → new app
3. Xcode → **Product → Archive** → upload to App Store Connect
4. Enable TestFlight, then submit for review
5. Privacy policy: `https://royal-match-poker.onrender.com/privacy.html`

## npm scripts (`frontend/`)

| Script | Purpose |
|--------|---------|
| `npm run build:mobile` | Vite build with production API URL |
| `npm run cap:sync` | Build + sync **Android and iOS** |
| `npm run cap:sync:ios` | Build + sync iOS only |
| `npm run cap:open:ios` | Open Xcode (Mac only) |
| `npm run cap:run:ios` | Sync + run simulator/device (Mac only) |

## Layout

| Item | Location |
|------|----------|
| Capacitor config | `frontend/capacitor.config.ts` |
| Xcode workspace | `frontend/ios/App/App.xcworkspace` |
| Mobile env (API URL) | `frontend/.env.capacitor` |

## Notes

- Login & saves use the same Render backend as web/Android.
- Orientation is **portrait-only** (matches in-game portrait gate).
- AdMob / push need separate iOS unit IDs and APNs setup.
- Do not commit `Pods/`, `DerivedData/`, or `App/App/public/` — regenerated by sync / pod install.

Windows can `npm run cap:sync:ios` and edit the project in git; **building the `.ipa` requires a Mac**.
