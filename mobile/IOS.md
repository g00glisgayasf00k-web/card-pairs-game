# Royal Poker Match — iOS (App Store / TestFlight)

The game is already a Capacitor app. Android lives in `frontend/android/`; iOS is the same React build wrapped in Xcode at `frontend/ios/`.

| Item | Location |
|------|----------|
| Capacitor config | `frontend/capacitor.config.ts` |
| Xcode project | `frontend/ios/App/App.xcworkspace` |
| Mobile web build (API → Render) | `frontend/.env.capacitor` |
| App ID | `com.royalmatch.poker` |
| API | `https://royal-match-poker.onrender.com` |

## What you need (Apple only)

1. **A Mac** — Xcode and Simulator only run on macOS. You can add/sync the iOS project on Windows; you cannot build or submit from Windows.
2. **Xcode 16+** from the Mac App Store.
3. **CocoaPods** — `sudo gem install cocoapods` (or Homebrew), then from `frontend/ios/App`: `pod install`.
4. **Apple Developer Program** — [developer.apple.com](https://developer.apple.com) ($99/year) for TestFlight and the App Store.

## Sync from this repo (any machine)

```powershell
cd frontend
npm install
npm run cap:sync:ios
```

That builds the Capacitor web bundle and copies it into `frontend/ios/`. On a Mac, Capacitor will also run `pod install` if CocoaPods is available.

## Open and run on a Mac

```bash
cd frontend
npm run cap:sync:ios
npm run cap:open:ios
```

In Xcode:

1. Select the **App** target → **Signing & Capabilities** → your Team.
2. Pick an iPhone simulator or a plugged-in device.
3. Press **Run ▶**.

Or from the terminal on a Mac:

```bash
npm run cap:run:ios
```

## App Store / TestFlight (short path)

1. Apple Developer → create an App ID matching `com.royalmatch.poker` (or change `appId` in `capacitor.config.ts` and re-sync).
2. App Store Connect → new app → bundle ID above.
3. In Xcode: **Product → Archive** → **Distribute App** → App Store Connect.
4. Upload, then enable **TestFlight** internal testing before public review.
5. Privacy policy URL (same as Play): `https://royal-match-poker.onrender.com/privacy.html`

Apple will ask for:

- Screenshots (6.7" and 6.1" iPhone at minimum)
- Age rating / content questionnaire
- Privacy nutrition labels (account, gameplay progress, ads if AdMob is on)

## Push notifications (APNs)

Same in-app badges as Android. Device push needs:

1. Firebase (or APNs cert) linked to `com.royalmatch.poker`.
2. Upload APNs key / certificate in Firebase or configure Capacitor Push on iOS.
3. Backend already sends via FCM when configured — iOS devices need an APNs-enabled Firebase iOS app.

## Ads

`@capacitor-community/admob` is already wired for native shells. Create an **iOS** AdMob app + unit IDs and set them in your native ad config (do not reuse Android unit IDs).

## npm scripts (from `frontend/`)

| Script | Purpose |
|--------|---------|
| `npm run build:mobile` | Vite build with production API URL |
| `npm run cap:sync` | Build + sync **Android and iOS** |
| `npm run cap:sync:ios` | Build + sync iOS only |
| `npm run cap:open:ios` | Open Xcode (Mac only) |
| `npm run cap:run:ios` | Sync + run on simulator/device (Mac only) |

## Notes

- Login & saves use the same Render backend as web/Android.
- After React changes, always `npm run cap:sync:ios` (or `cap:sync`) before archiving.
- Do not commit `Pods/`, `DerivedData/`, or `App/App/public/` — they are regenerated by sync / pod install.
